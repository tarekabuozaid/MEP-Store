## 2. الخلفية والمشكلة

النظام الحالي يعمل على ملف Excel `.xlsm` مع ماكرو VBA، وهو في الإنتاج الفعلي لمتاجر الظفرة. المشكلات التي تدفع نحو الترحيل:

| المشكلة | الأثر |
|---------|-------|
| لا حماية من التزامن | إذا فتح شخصان الملف يتعارضان في TxnID أو يخسران بيانات |
| لا عزل صلاحيات | كل من يفتح الملف يرى ويعدل كل شيء |
| لا تتبع لمن فعل ماذا | لا Audit trail فعلي |
| الأداء يتدهور | الملف يكبر مع الوقت، البحث الخطي O(n²) |
| الوصول صعب | يحتاج نسخ الملف أو فتحه من OneDrive |
| لا إشعارات | لا تنبيه تلقائي عند نقص المخزون |

---

## 3. الهدف

بناء نظام إدارة مخزون على Google يتيح:
1. **الأمين (Keeper):** إدخال معاملات مخزنه فقط عبر واجهة ويب بسيطة
2. **الأدمن (Admin):** رقابة كاملة على جميع المخازن، إدارة المستخدمين، تقارير شاملة
3. **النظام:** حماية البيانات من التزامن، سجل مراجعة كامل، حسب الموقع بدقة

---

## 4. المعنيون (Stakeholders)

| الدور | المسمى | الاهتمام الرئيسي |
|-------|--------|-----------------|
| **Admin** | مدير المخزون / مسؤول النظام | رؤية كاملة، تحكم كامل، تقارير |
| **Store Keeper** | أمين مخزن | إدخال معاملات سريعة، رؤية مخزنه فقط |
| **مراقب (اختياري)** | مدير يريد قراءة فقط | عرض التقارير بدون تعديل |

## 5. النطاق

### داخل النطاق ✅

- تسجيل أنواع المعاملات الأربعة: **استلام (Receipt)، صرف (Issuance)، تعديل (Adjustment)، تحويل (Transfer)**
- حماية المخزون من السالب للصرف والتحويل
- تحذير الحد الأدنى (غير مانع)
- توليد TxnID تلقائي وآمن
- التحويل كزوج صفين (OUT من المصدر + IN للوجهة)
- رصيد منعزل لكل موقع
- لوحة تحكم Admin بكل المخازن
- إدارة المستخدمين (إضافة/حذف/تعديل)
- سجل مراجعة (Audit Log) — من فعل ماذا ومتى
- تصدير Excel للتقارير
- تنبيهات الحد الأدنى على Dashboard

### خارج النطاق ❌

- تتبع تكلفة الوحدة (الأسعار) — مستبعد بقرار صريح
- تكامل مع أنظمة ERP خارجية
- إشعارات بريد إلكتروني / SMS تلقائية
- تطبيق موبايل native
- باركود / QR scanner
- طباعة مستندات

## 6. معايير النجاح

| المعيار | القياس | الهدف |
|---------|--------|-------|
| **تغطية المستخدمين** | % من أمناء المخازن مسجلين وفعّالين | 100% |

| **Audit trail** | كل معاملة مرتبطة بصاحبها | 100% |
| **رضا المستخدمين** | الأمين قادر يستخدم بدون تدريب مطوَّل | ≤ 30 دقيقة تعريف |

## 7. القيود والافتراضات

### قيود تقنية
- المنصة: **Google Apps Script** فقط (لا Node.js، لا Firebase)
- قاعدة البيانات: **Google Sheets** فقط
- المصادقة: **Gmail فقط** (لا Google Workspace مطلوب)
- النشر: Web App بإعداد `Execute as: Me` (حساب المالك)
- حصص Apps Script للحسابات الاستهلاكية: 6 ساعات/يوم تنفيذ، 100 مستخدم متزامن

### افتراضات
- جميع المستخدمين لديهم حساب Gmail نشط
- الأدمن يدير جدول `Users_Stores` يدوياً
- اتصال إنترنت متاح في جميع المخازن
- المتصفح الموصى به: Chrome أو Firefox
