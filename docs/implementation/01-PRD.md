# وثيقة متطلبات المنتج (PRD)
## Aldhafra IMS v2.0 — Google Apps Script Web App

---

## 1. شخصيات المستخدمين (User Personas)

### 1.1 الأدمن (Admin)
- **من هو:** مدير المخزون أو مسؤول النظام — شخص واحد أو عدد محدود
- **يستخدم من:** حاسوب مكتبي أو لابتوب
- **يحتاج:**
  - رؤية جميع المخازن في وقت واحد
  - إضافة/حذف/تعديل المستخدمين
  - مراجعة كل المعاملات التاريخية
  - إدارة قوائم الأصناف والمواقع
  - تصدير تقارير
- **توقعاته:** لوحة تحكم شاملة تعطيه "صورة واضحة" بدون تنقل كثير

### 1.2 أمين المخزن (Keeper)
- **من هو:** موظف في مخزن محدد — يسجل الحركات اليومية
- **يستخدم من:** تابلت أو موبايل أو حاسوب داخل المخزن
- **يحتاج:**
  - إدخال معاملة بسرعة (كود + كمية)
  - رؤية رصيد مخزنه
  - مراجعة تاريخ مخزنه
- **توقعاته:** نموذج بسيط يكمل فيه الإدخال في أقل من دقيقتين، بدون تعقيد

### 1.3 المراقب / القارئ (Viewer) — اختياري
- **من هو:** مدير يريد متابعة دون تعديل
- **يحتاج:** عرض فقط لأرصدة وتقارير مخزن أو أكثر

| US-07 | كأمين مخزن، أريد **مراجعة معاملاتي** السابقة مع إمكانية البحث | قائمة حركات مخزني مرتبة بالتاريخ |

## 3. المتطلبات الوظيفية (Functional Requirements)

### 3.1 وحدة المصادقة والتخويل

| المعرف | المتطلب |
|--------|---------|
| FR-AUTH-01 | النظام يرفض أي مستخدم غير مسجل في `Users_Stores` مع رسالة واضحة |
| FR-AUTH-02 | الأمين يرى فقط بيانات مخزنه المحدد في `Users_Stores.StoreCode` |
| FR-AUTH-03 | الأدمن يرى ويعدل كل شيء |
| FR-AUTH-04 | كل كتابة تسجل `UserEmail` من `Session.getActiveUser().getEmail()` |
| FR-AUTH-05 | تغيير صلاحية المستخدم يسري فوراً (لا cache للدور) |

### 3.2 وحدة المعاملات

| المعرف | المتطلب |
|--------|---------|
| FR-TXN-01 | أنواع المعاملات: `Receipt` \| `Issuance` \| `Adjustment` \| `Transfer` |
| FR-TXN-02 | كل معاملة تتطلب: نوع + تاريخ + موقع المصدر + صنف واحد على الأقل + كمية موجبة |
| FR-TXN-03 | صرف وتحويل: يُرفض لو الكمية > الرصيد المتاح في ذلك الموقع |
| FR-TXN-04 | تعديل: يُسمح مع تحذير لو النتيجة ستكون سالبة |
| FR-TXN-05 | تحويل يشترط: موقع وجهة ≠ موقع مصدر، وكلاهما موجودان في Locations |
| FR-TXN-06 | تاريخ أبعد من 30 يوماً: تحذير غير مانع |
| FR-TXN-07 | TxnID يُولَّد تلقائياً، فريد، بتنسيق `{PREFIX}-{YYYY}-{NNNN}` |
| FR-TXN-08 | التحويل ينشئ صفين في Stock_Movement: `-OUT` للمصدر و `-IN` للوجهة |
| FR-TXN-09 | الأمين لا يستطيع تعديل أو حذف معاملة محفوظة |
| FR-TXN-10 | حماية التزامن: LockService.getScriptLock() قبل أي كتابة |

### 3.3 وحدة المخزون

| المعرف | المتطلب |
|--------|---------|
| FR-STK-01 | الرصيد = مجموع كميات Receipt - مجموع كميات Issuance + مجموع Adjustment **لنفس الصنف والموقع** |
| FR-STK-02 | Adjustment الموجبة تضيف للرصيد، السالبة تطرح |
| FR-STK-03 | التحويل: الرصيد ينخفض في المصدر ويرتفع في الوجهة |
| FR-STK-04 | الرصيد يُحسب لحظياً من Stock_Movement (لا cached values) |
| FR-STK-05 | تحذير الحد الأدنى يظهر في Dashboard وعند الإدخال |

### 3.4 وحدة التقارير (Admin)

| المعرف | المتطلب |
|--------|---------|
| FR-RPT-01 | دفتر الحركات مع فلاتر: تاريخ من/إلى + موقع + نوع + كود الصنف |
| FR-RPT-02 | تصدير النتائج المفلترة كملف Excel (.xlsx) |
| FR-RPT-03 | Dashboard يُحدَّث عند كل فتح (لا auto-refresh) |
| FR-RPT-04 | سجل المراجعة (Audit Log) قابل للبحث |

### 3.5 وحدة إدارة المرجعية (Admin)

| المعرف | المتطلب |
|--------|---------|
| FR-REF-01 | الأدمن يضيف/يحرر أصناف في Master_Items |
| FR-REF-02 | الأدمن يضيف/يحرر مواقع في Locations |
| FR-REF-03 | الأدمن يضيف/يحذف مستخدمين في Users_Stores |
| FR-REF-04 | الكود الفريد للصنف والموقع لا يتغير بعد الإنشاء (يمكن إضافة فقط) |

## 4. المتطلبات غير الوظيفية (Non-Functional Requirements)

### 4.1 الأداء

| المتطلب | القيمة |
|---------|--------|
| وقت استجابة حفظ المعاملة | < 5 ثوانٍ |
| وقت تحميل أي شاشة | < 3 ثوانٍ |
| الحد الأقصى لعدد المستخدمين المتزامنين | 20 مستخدماً (في نطاق Apps Script) |
| حجم Stock_Movement قبل الأرشفة | 50,000 صف |

### 4.2 الأمان

| المتطلب | التفاصيل |
|---------|---------|
| المصادقة | Gmail OAuth عبر `Session.getActiveUser()` |
| التخويل | فحص مقابل `Users_Stores` في كل طلب |
| عدم التعديل المباشر | المستخدمون لا يملكون صلاحية Edit على Google Sheets المركزي |
| OAuth token | لا تُمرَّر للعميل أبداً |
| LockService | يمنع تعارض الكتابة |

### 4.3 قابلية الاستخدام

| المتطلب | التفاصيل |
|---------|---------|
| دعم الموبايل | الواجهة تعمل على شاشة 360px وما فوق |
| اللغة | العربية كلغة رئيسية، الأكواد التقنية بالإنجليزية |
| إمكانية التعلم | أمين مخزن جديد يستطيع تسجيل معاملة بعد 30 دقيقة تعريف |
| رسائل الخطأ | واضحة وتشير للحل المطلوب |

### 4.4 الموثوقية

| المتطلب | التفاصيل |
|---------|---------|
| Uptime | 99.5% (مرتبط بـ Google uptime) |
| النسخ الاحتياطي | Google Sheets يوفر تلقائياً version history |
| معالجة الأخطاء | كل خطأ يُسجَّل في Audit_Log ولا يُكسر الجلسة |

## 5. القيود والتبعيات

- لا تنفيذ خارج Google Apps Script (قرار نهائي)
- حسابات Gmail يجب أن تكون نشطة لكل مستخدم
- الأدمن يدير `Users_Stores` يدوياً (لا self-registration)
- حصص Apps Script: انظر [Google Quotas](https://developers.google.com/apps-script/guides/services/quotas)
